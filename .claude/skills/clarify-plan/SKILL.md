引数で渡された計画ドキュメント（`docs/plans/` 配下のMarkdownファイル）を読み込み、要求仕様を完成させる。

## 使い方

```
/clarify-plan docs/plans/plan_#1.md
```

引数省略時は `docs/plans/` 内のファイル一覧を表示して選択を促す。

## 手順

1. 指定されたプランファイルを読み込む
2. 「関連ファイル」セクションに列挙されたファイルをすべて読み込む
3. 実装に必要な情報で不明な点をリストアップする
4. **1問1答形式**でユーザーに質問する（一度に複数質問しない）
5. 回答を受け取るたびに、プランファイルの該当箇所を更新する
6. 不明点がなくなったら「要求仕様が完成しました」と伝える（実装は行わない）

## ルール

- 実装は絶対に行わない。仕様の明確化のみ。
- ユーザーの回答後は必ずプランファイルを更新してから次の質問をする。
- 技術的な詳細（どのFirestoreドキュメントを使うか等）はプランの「期待する動作・変更内容」または「関連ファイル」「制約・注意事項」セクションに追記する。
- 完了条件の `[ ]` に具体的な検証項目を記入する。
